LOCKSS Software Releases

LOCKSS Release Notes

Overview

Platform Releases: OpenBSD has a well-developed release process that produces a release every 6 months.  We aim to do a platform release ( a "new CD") about 6 weeks after each new OpenBSD release.  The changes to the LOCKSS platform appliance that are included with each such release are determined about a month before the OpenBSD release,  when the exact set of OpenBSD changes are clear.  We expect that the next such list will be generated in October 2004,  ready for a probable 3.6 OpenBSD release in November.  This allows 10 weeks for development and internal pre-release testing.  At present the only significant user-visible candidate for the next list is supporting machines with no floppy. There are many candidate cleanup and performance fixes.

Daemon Releases: The LOCKSS team has a release and test process that produces a new daemon every 6 weeks for release to and automatic download by the deployed LOCKSS peers.  The exact set of changes to be included in each 6-weekly release is pretty firm 4 weeks before the release and is frozen 2 weeks before to allow for testing.

Released Daemons

Daemon 1.4.6 (released 10/7/04)

  • further support for downloadable plug-ins
  • URL canonicalization (for IOP and HighWire)
  • define AU by issue ranges
  • refactoring of plug-in code
  • LockssApp pulled out of LockssDaemon
  • Crawler will not follow cross-host redirects without checking for permission statement on new host
  • Crawl permission check moved out of plug-in framework, into crawler
  • Improvements to handling of polls in which all caches have damage
  • removed need for org.lockss.plugin.xmlPlugins configuration setting
  • list of agreeing caches exported to UI

Daemon 1.3.6 (released 8/11/2004)

  • addition of the audit proxy port
  • plug-ins are now downloadable separately
  • new xml prop files are supported
  • UI comes up with warning until all AUs are started
  • tiny UI comes up when daemon is unable to reach prop server
  • initial cut of alerts framework

Future Releases

Daemon 1.5 (freeze late October, release mid November)

Meant to be a stability release, we're trying to keep new features to a minimum (except perhaps for UI fixes)
  • refinement of the Alerts framework
  • UI for configuration of Alerts
  • further UI improvements for adding many AUs
  • improving code test coverage
  • general bug fixes
  • ground work for the new protocol

Deamon 1.6 (freeze early December release mid January 2005)

Feature release, hopefully with some portion of the new protocol.
  • OAI crawling
  • initial version of the new protocol (may not be complete)
  • "Cache everything I have access to" feature in UI
